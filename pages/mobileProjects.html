Networking Project

This was a project for a networks class that covered design concepts and implementation features of computer networks. It was a three part project that spanned the entire semester. The idea behind it was to create a rudimentary internet using simulated nodes derived from a 'skeleton project'. The first part of the project was flooding and neighbor discovery. The second was routing tables that used dijkstra's algorithm. The third part was packet reliability with TCP. Once all three were finished, the program should be able to send a string of data from one node in the network to any other node that was connected.

UCM Course Notification

This program was created to notify users when a class they have been trying to sign up for has an open seat. There are two parts to the program, the web scraper and the Discord bot. The web scraper reads information about every class offered that semester and keeps track of how many seats are available for each class. If the number of seats increased or decreased it would add it to the update list.

The Discord bot is designed to be able to be able to serve multiple groups and multiple courses for each group. Users are able to send commands to request updates on specific classes using the course ID number. When the bot receives the update list, it will check if users are requesting any of the classes in the list and send a notification.

EyeBook Reader

This was a project for a Human-Computer-Interaction course. I worked in a team with three other classmates. We were tasked with creating an interface that would allow people with disabilites or missing limbs to more easily interact with an e-reader. Our solution was to use eye tracking to allow scrolling of the page if you look at the top or bottom of the page. I was tasked with implementing the eye tracking onto our website. We used a library called WebGazer to track the eye movements of the user. Our finished product allowed the user to upload a pdf or txt document and scroll up and down when looking at the top and bottom of the screen.

Latency Graph

This program was created to display the ping/latency currently being experienced and keeps a moving list of the last 100 pings. It was made using windows cmd ping command sending pings to google to get latency from their servers. It also includes a red line at 100 on the y axis so you can judge at a glance what your latency is.
