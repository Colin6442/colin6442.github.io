Senior Capstone

This was a program created for Western Digital as part of my senior project. I worked with three other students on this semester long project. I had the role of team lead, which consisted of delegating tasks within the team and communicating with the representatives at Western Digital. Once we were in contact with the representatives, we negotiated down the scope of the project since we only had one semester to complete the project. Networking Project

This was a project for a networks class that covered design concepts and implementation features of computer networks. It was a three part project that spanned the entire semester. We worked on implementing our own rudimentary internet using simulated nodes created from a 'skeleton project'. We worked on flooding and neighbor discovery, into creating routing tables making use of dijkstra's algorithm. We then implemented packet reliability with TCP. We then built a chat program that could run on our simulated internet. This was the most difficult class and project I worked on in college.

SuperHexagon Bot

This was a personal project I worked on to get more experience with image processing. The purpose of the game "SuperHexagon" is to pilot the player away from incoming walls, so I created a program that would be able to do that. It takes a screenshot every couple of milliseconds and analyzes the state of the game. Once it computes where all of the walls are, an algorithm determines the best action for the player to take. UCM Course Notification

This program was created to notify users when a class they have been trying to sign up for has an open seat. There are two parts to the program, the web scraper and the Discord bot. The web scraper reads information about every class offered that semester and keeps track of how many seats are available for each class. If the number of seats increased or decreased it would add it to the update list.

The Discord bot is designed to be able to be able to serve multiple groups and multiple courses for each group. Users are able to send commands to request updates on specific classes using the course ID number. When the bot receives the update list, it will check if users are requesting any of the classes in the list and send a notification.
